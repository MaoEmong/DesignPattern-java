# 템플릿 메서드 패턴 정리 (ex05 기반)

## 1. ex05에서 보이는 핵심 구조
- `Teacher` 추상 클래스가 `수업하기()` 메서드로 수업 흐름을 고정한다. (`src/ex05/teacher/Teacher.java`)
- `수업하기()` 내부 순서:
  1. `입장하기()`
  2. `출석부르기()`
  3. `강의하기()`
  4. `퇴장하기()`
- 이 중 `강의하기()`만 `protected abstract`로 선언되어 하위 클래스가 구현한다.
- `JavaTeacher`, `PythonTeacher`, `HtmlTeacher`는 `강의하기()`를 각자 다르게 구현한다. (`src/ex05/teacher/*.java`)
- `App`은 `Teacher` 타입으로 객체를 받아 `수업하기()`만 호출한다. (`src/ex05/App.java`)

## 2. 왜 템플릿 메서드 패턴인가
템플릿 메서드 패턴은 "상위 클래스에서 알고리즘의 전체 구조(템플릿)를 정의하고,
일부 단계만 하위 클래스가 바꾸도록" 하는 패턴이다.

ex05에서는 `수업하기()`가 템플릿 메서드이며,
공통 절차(입장/출석/퇴장)는 고정하고,
가변 절차(강의 내용)만 하위 클래스에 위임하고 있다.

## 3. 이 예제의 장점
- 수업 전체 절차가 한 곳(`Teacher`)에 모여 있어 일관성이 유지된다.
- 강의 과목이 늘어나도 `강의하기()`만 추가 구현하면 된다.
- 클라이언트(`App`)는 구체 클래스 차이를 몰라도 동일하게 `수업하기()`를 호출할 수 있다.

## 4. 역할 매핑
- AbstractClass: `Teacher`
- Template Method: `수업하기()`
- Primitive Operation(재정의 지점): `강의하기()`
- ConcreteClass: `JavaTeacher`, `PythonTeacher`, `HtmlTeacher`

## 5. 한 줄 요약
ex05는 "공통 흐름은 부모가 고정하고, 달라지는 핵심 단계만 자식이 구현"하는 템플릿 메서드 패턴의 전형적인 예제다.
